# Megalodon v9 — Fleet Matrix

Provider/model assignments per lane. Operator-overridable.

## Defaults

| Lane      | Model       | Why                                          | Cadence |
|-----------|-------------|----------------------------------------------|---------|
| AUDIT     | sonnet-4.6  | Observation/synthesis — cheap model OK       | 5m      |
| ARCHITECT | opus-4.7    | Heavy reasoning, SPEC drafting               | 5m      |
| BACKEND   | opus-4.7    | Code work                                    | 3m      |
| FRONTEND  | opus-4.7    | Code work                                    | 3m      |
| TEST      | opus-4.7    | Code work + e2e debugging                    | 3m      |
| META      | sonnet-4.6  | Observation                                  | 7m      |

These defaults are encoded both in `scripts/fleet_select.py` (model) and
`scripts/gen_lane_launches.py` (cadence). Update both if defaults shift.

## Per-mission overrides

Drop `<mission>/.scratch/fleet-matrix-override.json`:

```json
{
  "lanes": {
    "AUDIT": {"model": "haiku-4.5"}
  }
}
```

The override file is consulted by `scripts/fleet_select.py:select(lane, mission_dir)`.
Unknown lanes fall back to `opus-4.7`. Malformed JSON is ignored (silent
fallback to baked-in defaults).

## Why this is informational, not enforced

The matrix is operator-facing guidance. Workers don't need to consult it at
runtime — the lane-bound launch files generated by
`scripts/gen_lane_launches.py` already encode the per-lane model hint. The
matrix is the human's "what should I run where" reference.

## When to override

- **Budget constraint** — drop builder lanes to sonnet-4.6 when burning
  through opus-4.7 quota.
- **Capability test** — flip observer lanes to opus-4.7 to test whether the
  cheaper model is genuinely sufficient.
- **Mission-specific** — security-audit missions might pin AUDIT to opus-4.7
  even though synthesis usually doesn't need it.

## Cross-reference

- `scripts/fleet_select.py` — selector with override precedence.
- `scripts/gen_lane_launches.py` — codegen embeds the per-lane MODEL_HINT.
- `scripts/launch_fleet.sh` — emits the per-lane invocations.
- `launch.md` — RULE 15 (queue), RULE 16 (watchdog), §6.X PRE-CLASSIFY, §6.Y
  INTENT-EXPIRED for the worker-side discipline.
